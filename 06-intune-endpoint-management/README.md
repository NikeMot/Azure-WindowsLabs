# 06 — Intune Endpoint Management

### Purpose

This topic covers modern Windows endpoint management with Microsoft Intune, especially Windows Autopilot, device enrolment, compliance, configuration, application deployment, update management, endpoint security and troubleshooting.

### Focus Areas

* Windows Autopilot
* Autopilot device preparation
* Enrollment Status Page
* Windows out-of-box experience customisation
* device enrolment
* Microsoft Entra joined and hybrid joined device models
* compliance policies
* configuration profiles
* application deployment
* device actions and sync
* endpoint troubleshooting
* Conditional Access interaction
* Windows Update for Business
* Windows Autopatch
* BitLocker policy and recovery key handling
* Windows Hello for Business
* Windows LAPS
* Intune Endpoint Privilege Management
* security baselines
* modern workplace operations

### Example Labs

* Enrol a Windows device into Intune
* Configure a Windows Autopilot deployment profile
* Compare Autopilot with traditional imaging
* Configure Enrollment Status Page behaviour
* Configure compliance and investigate non-compliance
* Deploy an application to a test device group
* Configure Windows Update for Business update rings
* Explain where Windows Autopatch fits in enterprise endpoint operations
* Configure BitLocker policy and validate recovery key handling
* Investigate a failed or blocked device sign-in caused by compliance state
* Design a least-privilege endpoint admin model using Windows LAPS and Endpoint Privilege Management concepts
* Document endpoint support evidence

### Production Notes

Endpoint management labs should always consider:

* licensing and tenant assumptions
* device join model
* user-driven versus admin-driven provisioning
* rollback and device reset paths
* least privilege
* user disruption
* support desk troubleshooting evidence
* monitoring and reporting
* security baseline impact
