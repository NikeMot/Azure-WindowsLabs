# 09 — SRE-Style Hybrid Operations Capstone

### Purpose

This topic combines Windows Server, Active Directory, PowerShell, IIS/application support, Entra ID, Intune, Azure, monitoring, backup, incident response and SRE-style reliability practices into final operational scenarios.

The capstone should simulate operating a production service, not just configuring infrastructure.

### Focus Areas

* service ownership
* dependency mapping
* Windows/IIS application reliability
* AD and DNS dependencies
* Azure and hybrid monitoring
* SLI and SLO thinking
* alerting and incident response
* rollback and recovery planning
* post-incident review
* reliability improvement backlog
* toil reduction through PowerShell automation
* stakeholder communication
* documentation and runbooks
* verification evidence

### Example Labs

* Operate a Windows/IIS service with AD, DNS, storage and Azure/hybrid dependencies
* Investigate a simulated IIS/application outage during an on-call shift
* Define SLIs and SLOs for a Windows-hosted internal service
* Collect monitoring evidence from Windows logs, IIS logs, PowerShell and Azure-style telemetry
* Produce an incident timeline, stakeholder update and post-incident review
* Build a PowerShell health check or evidence collection script
* Create a reliability improvement backlog for the service
* Complete the final SRE-style production incident drill

### Required Capstone Artefacts

* service overview
* dependency map
* SLI/SLO definition
* monitoring evidence
* incident timeline
* technical triage notes
* stakeholder update
* recovery or rollback plan
* post-incident review
* automation artefact
* reliability improvement backlog
